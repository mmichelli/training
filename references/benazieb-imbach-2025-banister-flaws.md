# Statistical Flaws of the Fitness-Fatigue Sports Performance Prediction Model

**Authors:** O. Benazieb, Y. Weldegebriel, T. Méline, F. Imbach (+ others)
**Journal:** Nature Scientific Reports
**Year:** 2025
**Source URL:** https://www.nature.com/articles/s41598-025-88153-7
**Fetched via Google Scholar:** 2026-05-14 — full text behind a Nature login redirect; the metadata + headline findings below come from Scholar's indexed summary.

## What we use this for

Justifies *not* using TSB / Banister "form" as a tapering oracle in `principles.md` §4 ("What we explicitly don't track"). We still use a smoothed acute-vs-chronic load ratio (ACWR) as a coarse load proxy, but we treat the fatigue/form decomposition as poorly identifiable rather than load-bearing.

## Key findings (from indexed summary)

- The classic Banister fitness-fatigue model is **ill-conditioned**: the fitness and fatigue parameters are not individually identifiable from typical training-data records.
- Adding the fatigue term produces **overfitting** without improving genuine predictive power beyond the fitness term alone.
- The model can look like it fits past data while being unreliable for forward prediction (tapering/peak-day form estimates).

## How this constrains our claims

- ✓ Defensible: "Don't trust TSB as a peak-day predictor."
- ✓ Defensible: "Acute:chronic load ratio is still useful as a coarse injury-risk proxy" — that's a separate model from FFM.
- ⚠ Caveat: I have not read the full paper. The headline framing here matches Scholar's indexed summary; if the actual paper qualifies the claim (e.g., FFM works on certain populations with enough data), update this note.
